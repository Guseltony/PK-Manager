# 🏗️ Execution Architecture (Projects) — Technical Review

The Projects module is the "Mid-Range Bridge" between long-term Ambitions and daily Tasks. It provides the structured context required for complex execution.

---

## 🔍 System Analysis

### 1. The "Blueprints" Logic
Projects act as the template for reality. The current architectural health system (stalled days tracking) is critical for preventing "Project Drift."
*   **Aesthetic Shift**: Moving from "Soft/Pill" design to **"Brutalist/Square"** for the execution layer.

### 2. Design Adjustments (Brutalist Lockdown)
Based on the specific request for the "Control Board" and "Live Blueprints" sections:

| Feature | Change | UX Impact |
| :--- | :--- | :--- |
| **Global Boxes** | `p-x: 2` | Increases horizontal density on mobile. |
| **Control Board** | `p-x: 1` | Maximizes information width for complex data. |
| **Border Corners**| `rounded-none` | Creates a distinct, technical "Industrial Blueprint" aesthetic. |

## 💡 Improvement Roadmap

*   **Task-to-Project Routing**: Ensure that completing 100% of tasks in a project triggers a "Review Parent Ambition" prompt.
*   **Gap Analysis**: Automate the detection of projects that have no tasks scheduled in the next 7 days (Predictive Stalling).
